01ما الذي يقوم به النظام

المالك يفتح /chat ويبدأ محادثة عربية حرّة. الـ Chat Agent (Sonnet 4.6) يستخرج حتى ١٠ حقول من جملة واحدة — اسم المحل، نوعه، تواريخ الرخص، عدد الطفايات، نظام التهوية، شهادات الموظفين الصحية، اعتماد اللوحة.

بعد ١٦ سؤال، الـ Project Orchestrator يطلق ١٢ ايجنت في موجات متتالية، كل واحد متخصص في جهة حكومية أو دور تنسيقي، ويتعاونون عبر AgentBus حقيقي بـ inbox مستقل لكل ايجنت.

النتيجة: داشبورد يومي فيه «صحة الرخص ٧٥٪، البلدية تنتهي خلال ١١ يوم، شهادات صحية ناقصة لموظفَين» + تذكيرات بريدية تلقائية قبل ٣٠ يوم من أي تجديد.

الفئات المدعومة

  • كوفي / مقهى — civil_defense + municipality + sfda
  • مطعم — civil_defense + municipality + sfda + moh
  • بقالة / سوبرماركت — civil_defense + municipality + sfda
  • مغسلة ملابس — civil_defense + municipality
  • صالون — civil_defense + municipality + moh

02القرارات التي يتخذها كل وكيل

الوكيل المحرّك القرار
Chat Sonnet 4.6 أي حقل ينقص، هل يثق بالاستخراج، متى تكتمل المقابلة
Orchestrator Code جدولة الموجات، dependencies، انتقال phase
Research Web Search تصفية التحديثات التنظيمية، أي ايجنت يحتاج إخطار
MCI LLM + Tools صلاحية السجل، موعد التجديد، أنشطة مفقودة
Civil Defense LLM + Tools عدد الطفايات حسب المساحة والنشاط، فحص دوري، رسالة hasKitchen للبلدية
Municipality LLM + Tools اعتماد اللوحة، رخصة المطبخ (لو وصلت رسالة من Civil Defense)، تكلفة بلدية
SFDA LLM + Tools هل يحتاج ترخيص غذائي، صلاحيته، تتبّع تبريد
MOH LLM + Tools كم موظف يحتاج شهادة صحية، هل العدد كافٍ
MOHR + GOSI LLM + Tools تقدير نطاق السعودة (أخضر/أصفر/أحمر)
ZATCA LLM + Tools هل تجاوز حد VAT (٣٧٥ ألف)، الفوترة الإلكترونية
Analysis Sonnet 4.6 تقييم الفجوات (critical/high/medium/low)، topWarnings، narrative عربي

03وكيل واحد أم متعدد؟

متعدد — ١٢ ايجنت في طبقتين:

  • طبقة التنسيق (٥): orchestrator · chat · research · analysis · report
  • طبقة المتخصصين (٧): mci · municipality · civil_defense · sfda · moh · mohr_gosi · zatca

دليل أن الـ A2A ليس زخرفة

CivilDefenseAgent يصدر outbox فيها hasKitchen: true → Municipality يقرأ inboxه → يضيف «رخصة المطبخ التجاري». بدون هذي الرسالة الرخصة لا تظهر إطلاقاً في تقرير المطعم.

الكود في src/agents/specialists/civil-defense-agent.ts سطر 65. حذف الـ outbox يكسر هذا السلوك مباشرة — يعني الرسالة تحدّد المخرج النهائي وليست logging.
